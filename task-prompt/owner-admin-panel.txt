Extend the existing FaithStack Laravel 13 multi-tenant CMS by implementing a Platform Super Admin Panel (SaaS Owner Panel).

This panel is separate from tenant dashboards and is accessible at:

👉 http://faithstack.test/admin (no subdomain)

🎯 Goal

Create a centralized admin system where the platform owner can:

Manage all tenants
Create and onboard new tenants
Control subscriptions
Monitor overall platform usage

This panel must be completely independent from tenant-based routing and must bypass tenant scoping.

🧱 Core Requirements
1. Super Admin Authentication
Implement authentication for platform admins
Option 1: Separate guard (recommended)
Option 2: Use users table with role = superadmin
users table addition:
role (admin, superadmin)
Requirements:
Only superadmins can access /admin
Block normal tenant users
2. SuperAdmin Middleware

Create middleware:

SuperAdminMiddleware
Ensures user is authenticated AND role = superadmin
Must bypass tenant resolution
Should NOT rely on subdomain
3. Separate Routing Structure

Define routes outside tenant middleware group:

Route::prefix('admin')->middleware(['auth', 'superadmin'])->group(function () {
    Route::get('/', [SuperAdminDashboardController::class, 'index']);
    Route::resource('tenants', TenantController::class);
});

Ensure:

These routes do NOT use tenant_id filtering
These routes operate globally
