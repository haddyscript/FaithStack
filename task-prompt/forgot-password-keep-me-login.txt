Backend Implementation Prompt

    Prompt: Develop a secure, production-ready backend implementation for "Forgot Password" and "Keep Me Signed In" (Remember Me) features for a multi-tenant web application.

    1. Forgot Password Workflow:

        Request Logic: Create an endpoint to handle password reset requests. It must validate the email, check if the user belongs to the current tenant, and generate a unique, time-limited (e.g., 60 mins), one-time-use reset token.

        Security: Ensure the token is hashed in the database. Implement rate limiting (throttling) to prevent email enumeration attacks.

        Email Integration: Design the logic to trigger a transactional email containing a secure link (e.g., /reset-password/{token}).

        Reset Logic: Create a second endpoint to validate the token and update the password using secure hashing (Bcrypt/Argon2).

    2. "Keep Me Signed In" (Remember Me) Logic:

        Token Strategy: Use a "Long-lived Selector and Validator" pattern (standard for production security).

        Cookie Management: On successful login with the checkbox checked, generate a secure, HttpOnly, SameSite=Lax, and Secure cookie.

        Persistence: Store a hashed version of the validator in the database linked to the user ID.

        Authentication Middleware: Update the auth middleware to check for this cookie if the session has expired, re-authenticating the user automatically if the tokens match.

    3. Multi-Tenancy Constraints:

        Ensure all queries are scoped to the tenant_id to prevent cross-tenant account access or password resets.
