Task: Refactor Navigation Management UI for Drag-and-Drop UX
Link : http://church1.faithstack.test/admin/navigation
I am building a "Navigation Menu" manager. Currently, it uses a simple two-column layout: a list of items on the left and a creation form on the right. I want to modernize this into a highly functional, interactive builder.

Current Stack & Context:

    Left Side: Displays "Current Menu Items" with Label, URL, Order, and a "Remove" button.

    Right Side: A form to "Add Menu Item" (Label, URL, Order).

Enhancement Requirements:

    Drag-and-Drop Reordering:

        Replace the manual "Order" input field with a draggable list.

        Each menu item should have a "drag handle" icon (six dots/grip).

        Suggest a library (like SortableJS or dnd-kit) or a clean Tailwind-based implementation for the UI state.

    Enhanced List Items:

        Redesign the list items as slim, horizontal cards.

        Show the URL as a "badge" or muted text.

        Add an "Edit" button alongside "Remove" so users don't have to delete an item just to change its name.

    Hierarchical Support (Nested Menus):

        Modify the UI to allow for nested/sub-menu items (indentation). If an item is dragged slightly to the right, it should visually nest under the item above it.

    Form Optimization:

        Since "Order" is now handled by dragging, remove the "Order" input from the "Add Menu Item" form.

        Add a "Link Type" selector (e.g., Internal Page vs. External URL) to help users auto-complete the URL field.

    Visual Feedback:

        Add an "Empty State" illustration or message if no menu items exist.

        Use a subtle "Ghost" or "Placeholder" style when an item is being dragged to show where it will land.

Output Style: Provide the HTML/Blade/JSX and CSS. Use a clean, modern aesthetic that matches a professional CMS.
